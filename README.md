# Taiga - ZIT 2026

**Zadanie:** Stworzenie prostej strony głównej dla systemu zarządzania praktyk zawodowych.

---

1. Znajdź swoje zadanie w Taidze. Przypisz je do siebie
2. Sforkuj to repozytorium (przycisk Fork w prawym górnym rogu)
3. Sklonuj swojego forka na dysk lokalny (`git clone <link_do_forka>`)
4. Utwórz nowy branch (`git checkout -b <nazwa_brancha>`)
5. Wykonaj swoje zadanie zgodnie z instrukcjami - wklej kod w odpowiednie miejsce w projekcie
6. Commit i Push (`git add .` > `git commit -m "TG-<numer_zadania> #<status>` > `git push origin <nazwa_brancha>`, gdzie `#<status>` to `#done`)
    - `"TG-<numer_zadania> #<status>"` NIE zmieni statusu User Story w Taidze, ponieważ pracujemy na Pull Requestach, a nie na commitach bezpośrednio do repozytorium (aby uniknąć konfliktów). To prowadzący (osoba scalająca commity) musi odpowiednio nazwać PR. Jednak proszę o trzymanie się nazewnictwa aby prowadzący wiedział, jaką nazwę commita przepisać oraz aby zapoznać się z nazewnictwcem Taigi :)
    - Numer zadania Taigi (`TG-<numer_zadania>`) jest widoczny obok nazwy User Story w Taidze
7. Utwórz Pull Request z twojego forka do main głównego repozytorium
